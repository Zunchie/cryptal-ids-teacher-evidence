# Selected Decision Records Summary

The source repo evidence shows many docs-only contracts and decision records. These records were used to define what the next allowed action was, what was out of scope, and what validation was required.

Decision records helped the project avoid vague automation. Instead of allowing the system to keep acting indefinitely, each step had to be justified and reviewed.

Examples from PR and issue titles include result review contracts, architecture replacement contracts, audit result decisions, semantic failure decisions, and controlled research decisions.
