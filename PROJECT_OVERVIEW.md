# Project Overview

**Project name:** Cryptal Quant Agent / ProjectProxy  
**Course:** Independent Directed Studies 11/12  
**Status:** Unfinished private work-in-progress

The project started as an ambitious automation idea around quantitative crypto research. As the work developed, the main learning shifted toward ProjectProxy: a controlled automation and evidence system designed to keep actions bounded, reviewable, and human-gated.

The project attempted to organize planning, validation, repair, and next-step control through GitHub issues, pull requests, decision records, and checks. Instead of treating automation as simply “make the computer do more,” the project focused on when automation should stop, how it should explain itself, and how a human can verify each step.

The technical challenge was not only coding. It included workflow design, control-plane design, documentation discipline, failure analysis, and privacy-safe evidence collection.
